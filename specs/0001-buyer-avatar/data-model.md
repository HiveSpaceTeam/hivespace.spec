# Data Model: Buyer Avatar

## Buyer Profile

Owned by UserService.

| Field | Meaning | Validation |
|---|---|---|
| `userId` | Authenticated user identity | Derived from current JWT subject |
| `userName` | Existing profile username | Existing profile validation |
| `fullName` | Existing profile display name | Existing profile validation |
| `avatarFileId` | Media file assigned as the user's current avatar | Non-empty GUID string for custom avatars |
| `avatarUrl` | Last successfully resolved public avatar URL | Nullable only when no avatar has ever been resolved |

### State Transitions

| State | Trigger | Result |
|---|---|---|
| No custom avatar | Buyer assigns uploaded file | `avatarFileId` set, `avatarUrl` remains null until processing completes |
| Pending avatar | Media processing succeeds | `avatarUrl` set from `MediaAssetProcessedIntegrationEvent` |
| Existing avatar | Buyer assigns a new file | `avatarFileId` replaced, previous `avatarUrl` remains visible until new processing completes |
| Pending avatar | Upload/processing fails | Previous `avatarUrl`, when present, remains the displayed avatar; UI shows failure or pending feedback without a broken image |

## Avatar Image

Owned by MediaService as a `MediaAsset`.

| Field | Meaning | Validation |
|---|---|---|
| `fileId` | Media asset identity | Generated by MediaService |
| `entityType` | Business use of the media asset | `user_avatar` for this feature |
| `entityId` | Owning user ID, when provided during confirmation | Current authenticated user ID |
| `contentType` | Browser-reported MIME type | JPEG, PNG, or WebP for buyer avatars |
| `fileSize` | Declared file size | Greater than 0 and not more than 5 MB |
| `publicUrl` | Display URL after processing | Produced by MediaService |
| `status` | Media lifecycle | Pending, uploaded, processed, or failed |

## Default Avatar

Frontend fallback display state, not a persisted domain entity.

| Field | Meaning |
|---|---|
| `src` | Empty/null when no custom avatar is available |
| `alt` | Localized accessible label for the user's avatar |

## Relationships

- One Buyer Profile references zero or one current Avatar Image by `avatarFileId`.
- MediaService owns the Avatar Image lifecycle and emits processing completion.
- UserService consumes media processing completion for `user_avatar`, matches event `fileId` to Buyer Profile `avatarFileId`, and stores the resolved `avatarUrl` for profile display.
